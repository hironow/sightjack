# S0012. Reference Data Management

**Date:** 2026-02-28
**Status:** Accepted

## Context

4つの CLI ツール (phonewave, sightjack, paintress, amadeus) は外部システム
(Linear, GitHub) と連携する。これらの外部システムが管理するデータの取り扱い
について統一方針が必要。

外部システムには以下の特性がある:

- **Linear**: 確定された issue 群が保存される単一の永続化レイヤー。全件再取得
  は API レート制限やデータ量の観点から非現実的。
- **GitHub**: コードは Git リポジトリに、レビューは Pull Request に保存される。
  PR 全件やレビューコメント全件の再取得は非現実的。

ツールがこれらの全データのコピーをローカルに保持すると:

1. データの鮮度が保証できない (stale copy)
2. ストレージ肥大化
3. 一貫性維持の複雑さ (eventual consistency の管理コスト)
4. 機密データの不必要な複製

## Decision

### 原則: Single Source of Truth (SSoT) の尊重

各外部システムを SSoT として扱い、ローカルには以下のみ保持する:

1. **参照 (references)**: ID, URL, タイトルなど最小限の識別情報
2. **要約 (summaries)**: 自然言語による要約テキスト
3. **カテゴリ (categories)**: 分類やタグなどのメタデータ
4. **意味的まとまり (semantic groups)**: 関連アイテムのグループ情報

### 禁止事項

1. 外部システムのデータ全件コピーをローカルに保持しない
2. 外部システムの内部 ID 体系に依存した独自インデックスを構築しない
3. ローカルコピーを SSoT として扱わない (常に外部システムが正)

### 許可事項

1. 操作に必要な最小限の参照情報をイベントストアに記録する
2. D-Mail 経由で受け取った要約・レポートをアーカイブに保存する
3. 直近の操作で使用した参照データをキャッシュする (TTL 付き)

### Git 管理下のデータ

参照データが Git 管理下に入る場合は S0003 (*.local.* パターン) に従い、
環境依存情報 (API トークン、ローカルパス) を分離する。

## Consequences

### Positive

- データの鮮度問題が発生しない (常に外部システムから取得)
- ストレージ使用量が最小限
- 機密データの不要な複製を防止

### Negative

- 外部システムが利用不可のとき、参照情報のみでの動作に制限される
- オフライン動作時は最後に取得した参照データに依存する

### Neutral

- 各ツールは外部システムへのアクセスパターンを自身で定義する
- キャッシュ戦略は各ツールの判断に委ねる
